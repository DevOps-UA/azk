azk help

Usage:
  azk agent (start|status|stop) [--no-daemon --child --reload-vm -q -l -h] [-v]...
  azk config (track-toggle|track-status) [-q -l -h] [-v]...
  azk doctor [--logo -q] [-v]...
  azk docker [-q -l -h] [-v]... [-- <docker-options>...]
  azk info [--no-color -q -l -h] [-v]...
  azk init [<path>] [--filename --force -q -l -h] [-v]...
  azk stop [<system>]  [-q -l -h] [-v]...
  azk start [<system>] [--reprovision --rebuild --open] [-q -l -h] [-v]...
  azk scale [<system>] [<to>] [-q -l -h] [-v]...
  azk vm (ssh|start|status|installed|stop|remove) [-q -l -h -f] [-v]... [-- <ssh-options>...]
  azk [help | -h | --help]
  azk [version | --version]

Options:
  -h, --help     Show this screen.
  --version      Show version.
  -v, --verbose  Sets the level of detail - multiple supported (-vv == --verbose 2) [default: 0].
  -l, --log      Sets a log level.
  -f, --force    Force mode.
  -q, --quiet    Never prompt [default: false].


commands:
  agent    Controls azk agent
  config   Controls azk configuration options
  docker   Alias for calling docker in 'azk' scope configuration
  doctor   Shows an analysis of `azk`'s health
  help     Shows help about a specific command
  info     Shows systems information for the current `Azkfile.js`
  init     Initializes a project by adding the file Azkfile.js
  logs     Shows logs for the systems
  reload   Stops all systems, run provision commands and starts them again
  restart  Stops all systems and starts them again
  scale    Scales (up or down) an instance of the system(s)
  shell    Initializes a shell with instance context, or runs a specified command
  start    Starts an instance of the system(s)
  status   Shows systems(s) instances status(es)
  stop     Stops an instance of the system(s)
  version  Shows azk version
  vm       Controls a virtual machine.

examples:

  $ azk agent start --daemon
  $ azk shell --image azukiapp/busybox
  $ azk shell --mount ~/Home:/azk/user -e HOME=/azk/user
  $ azk status [system_name]
  $ azk scale [system_name] 2
